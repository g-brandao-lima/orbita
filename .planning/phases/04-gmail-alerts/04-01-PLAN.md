    Implementar app/services/alert_service.py substituindo os stubs por codigo real:

    1. `compose_alert_email(signal, group)`:
       - Cria MIMEMultipart("alternative")
       - Subject: `[{signal.urgency}] {signal.signal_type} - {group.name}`
       - From: settings.gmail_sender, To: settings.gmail_recipient
       - Gera silence_token via generate_silence_token(group.id)
       - silence_url: `{settings.app_base_url}/api/v1/alerts/silence/{token}?group_id={group.id}`
       - Attach MIMEText(plain, "plain", "utf-8") e MIMEText(html, "html", "utf-8")
       - HTML template com urgency_colors dict (MAXIMA=#dc2626, ALTA=#ea580c, MEDIA=#ca8a04)
       - Corpo: grupo nome, rota (origin -> destination), datas, preco (R$ formatado), detalhes, link silenciar

    2. `send_email(msg)`:
       - `smtplib.SMTP_SSL("smtp.gmail.com", 465, timeout=30)`
       - server.login(settings.gmail_sender, settings.gmail_app_password)
       - server.send_message(msg)

    3. `generate_silence_token(group_id)`:
       - secret = settings.gmail_app_password.encode() (reusa senha como secret — single-user, aceitavel)
       - message = f"silence:{group_id}".encode()
       - hmac.new(secret, message, hashlib.sha256).hexdigest()[:32]

    4. `verify_silence_token(token, group_id)`:
       - expected = generate_silence_token(group_id)
       - hmac.compare_digest(token, expected)

    5. `should_alert(group)`:
       - Se group.silenced_until is None: return True
       - Se datetime.utcnow() > group.silenced_until: return True
       - Senao: return False

    6. Executar `python -m pytest tests/test_alert_service.py -v` — todos passam (GREEN).
    7. Executar `python -m pytest tests/ -x -q` — suite completa passa (sem regressao).
