# Phase 21.5: Sentry Observability - Context

**Gathered:** 2026-04-20
**Status:** Implemented (DSN configurado, pronto para producao)

<domain>
Instrumentacao de monitoramento de erros em producao via Sentry. Captura exceptions, request context, e performance traces (10% sample) sem enviar PII.
</domain>

<decisions>
- DSN via SENTRY_DSN no .env (e render.yaml como secret)
- send_default_pii=False (LGPD: sem IP, cookies, headers)
- user_id associado manualmente via bind_user_context no AuthMiddleware
- before_send scruba Authorization/Cookie/X-API-Key headers e chaves com "password/secret/token/api_key"
- traces_sample_rate=0.1 (10% requests) para caber em free tier 5k events/mes
- Desativado em pytest (check sys.modules + PYTEST_CURRENT_TEST)
- Desativado quando DSN vazio (dev local sem config)
- Endpoint /sentry-debug temporario em main.py para validar integracao (remover apos primeira confirmacao)
</decisions>
