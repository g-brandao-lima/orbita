  <action>
    Problema: `func.strftime("%w", FlightSnapshot.departure_date)` (linha 91) e `func.strftime("%Y-%m-%d %H", FlightSnapshot.collected_at)` (linha 151) sao funcoes SQL do SQLite. PostgreSQL nao tem strftime e crasha com ProgrammingError.

    Abordagem: Usar Python-side processing ao inves de SQL dialect-specific functions. Ambas as queries podem ser reescritas para funcionar em qualquer banco.

    1. **best_day (linhas 87-108)**: Em vez de agrupar por dia da semana no SQL com strftime, buscar os snapshots relevantes e calcular em Python:
       - Buscar todos os snapshots do grupo com origin/destination filtrados
       - Em Python, agrupar por `departure_date.weekday()` (0=Monday no Python, converter para nomes pt-BR)
       - Calcular media de preco por dia da semana
       - Filtrar dias com >= 3 snapshots
       - Retornar o dia com menor media

    2. **collection_count (linhas 148-155)**: Em vez de `func.count(func.distinct(func.strftime(...)))`, usar:
       - `func.count(func.distinct(func.date_trunc('hour', FlightSnapshot.collected_at)))` NAO funciona em SQLite.
       - Melhor abordagem: buscar collected_at distintos e agrupar em Python por hora, ou simplesmente contar distinct dates:
       - Usar `db.query(func.count(func.distinct(func.cast(FlightSnapshot.collected_at, Date))))` para contar dias distintos de coleta. Isso funciona em ambos os dialects.
       - Alternativa mais simples e portavel: `db.query(FlightSnapshot.collected_at).filter(...).all()` e contar horas distintas em Python com `set(ca.strftime("%Y-%m-%d %H") for ca in results)`.

    Solucao concreta para cada caso:

    **best_day (substituir linhas 87-108):**
    ```python
    best_day = None
    day_snaps = (
        db.query(FlightSnapshot.departure_date, FlightSnapshot.price)
        .filter(
            FlightSnapshot.route_group_id == group.id,
            FlightSnapshot.origin.in_(group.origins),
            FlightSnapshot.destination.in_(group.destinations),
            FlightSnapshot.departure_date.isnot(None),
        )
        .all()
    )
    if day_snaps:
        from collections import defaultdict
        day_prices = defaultdict(list)
        for snap in day_snaps:
            dow = snap.departure_date.weekday()  # 0=Monday
            day_prices[dow].append(snap.price)
        # Filter days with >= 3 samples, find cheapest
        day_names = ["Segunda", "Terca", "Quarta", "Quinta", "Sexta", "Sabado", "Domingo"]
        candidates = [
            (dow, sum(prices) / len(prices))
            for dow, prices in day_prices.items()
            if len(prices) >= 3
        ]
        if candidates:
            candidates.sort(key=lambda x: x[1])
            best_dow, best_avg = candidates[0]
            best_day = {"name": day_names[best_dow], "avg_price": round(best_avg, 0)}
    ```

    **collection_count (substituir linhas 148-155):**
    ```python
    collected_times = (
        db.query(FlightSnapshot.collected_at)
        .filter(FlightSnapshot.route_group_id == group.id)
        .all()
    )
    collection_count = len(set(
        ct[0].strftime("%Y-%m-%d %H") for ct in collected_times if ct[0]
    ))
    ```

    Notas:
    - Mover `from collections import defaultdict` para o topo do arquivo.
    - O `.strftime()` no collection_count e Python-side (no objeto datetime), NAO SQL-side. Funciona igual em SQLite e PostgreSQL.
    - day_names agora usa weekday() Python (0=Monday) ao inves de strftime("%w") SQLite (0=Sunday). Ajustar a ordem.
    - Manter a mesma interface de retorno: best_day como dict com "name" e "avg_price", collection_count como int.
